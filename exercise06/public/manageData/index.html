Here on these page you can manage your routes. That means that your be able to upload new routes, deleting old routes and renaming existing routes. Notice that every name for a route can only used once, so make sure that your new routes have a not used name or rename one old route. For the deleting and renaming are functions inside the table. If you want want to delete some routes, mark their checkboxes and hit the delete button. For renaming type a new name inside the box and hit the Ok button. For upload a new route use the upload a file button. The data must be stored as an json or geojson file and need to respect some formatting guidelines. Make sure that is only one object and not an array. These object should hold the three keys "type", "properties" and "geometry" in the first order. "type" need to have the value "Feature", while "properties" and "geometry" are objects. The "properties" need a key "name" with a unique value. You can also add extra key value pairs with additonal information to the propertiess object. In the geometry object your need to add a "type" key also. At these place the value must be "MultiLineString". Also your adding a key "coordinates". These coordinatess must follow a strict format. It must be an one element array, which contains an array with all points for your route. Each of these array elements need to be an array with to elements. These two elements are the coordinates of one point of the route. The coordinates must be in the format [lat,lon]. The coordinates must be in WGS84. Only if you can use these format the server can interprete the data.
